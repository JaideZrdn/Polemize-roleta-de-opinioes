## O que é Polemize?

**Polemize** é um app para criar conversas em uma roda de amigos, gerando discussões acaloradas sobre diversos temas a partir de polêmicas. Com o subtítulo de **"Roleta Russa das Opiniões"**, o app surgiu de uma discussão em grupo sobre como seria interessante ter uma ferramenta para debates divertidos e imprevisíveis.

### Como Funciona

Reúna seus amigos, insira os nomes deles, escolha os temas que poderão aparecer e deixe o Polemize fazer o resto. O app oferece dois modos de operação: \
<img height="1500" alt="Anuncio3" src="https://github.com/user-attachments/assets/43f510c5-c2ec-4da9-b8ee-44765ac607eb" />

#### Modo com IA
O app usa inteligência artificial para analisar as características dos jogadores, preferências e relacionamentos, criando controvérsias personalizadas. A IA decide automaticamente quem debate com quem e qual será o próximo tópico controverso, tornando cada jogo único, imprevisível e ainda mais divertido. \
<img height="1500" alt="Anuncio4" src="https://github.com/user-attachments/assets/0157d748-5df8-4c27-b6eb-3ae35d858849" />

#### Modo sem IA
O app seleciona aleatoriamente polêmicas de uma base de tópicos pré-definidos, garantindo debates divertidos e provocativos sem a necessidade de personalização por IA.

## Minha Contribuição no Projeto

Como desenvolvedor, fui responsável por aspectos técnicos do aplicativo:

### Desenvolvimento Técnico
- **Sistema de Alimentação de Dados**: Implementação do sistema de carregamento e gerenciamento das polêmicas via **JSON**
- **Navegação**: Desenvolvimento do sistema de navegação utilizando padrão **Coordinator** com **SwiftUI**
- **Arquitetura**: Organização do projeto seguindo **Clean Architecture** combinada com padrão **MVVM**, garantindo separação de responsabilidades e manutenibilidade
- **Gerenciamento de Estado**: Implementação da passagem de informações entre Views utilizando componentes nativos do **SwiftUI** com **ViewModels**

---

## Tecnologias Utilizadas

- **SwiftUI**: Framework para desenvolvimento da interface do usuário
- **Coordinator Pattern**: Gerenciamento de navegação e fluxo entre telas
- **JSON**: Estrutura de dados para armazenamento e carregamento das polêmicas
- **Clean Architecture + MVVM**: Organização do código com separação clara de camadas
